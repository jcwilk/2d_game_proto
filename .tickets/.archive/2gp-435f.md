# Implement centralized engine module for viewport and DisplayMode

Single module owns `Engine`: **`DisplayMode`**, resolution policy, **`suppressHiDPIScaling`**. **Normative:** **`.cursor/plans/project-implementation-deep-dive.md`** §D.1.

## Design

Export width/height constants or factory; **pick one** policy: fixed logical resolution **or** native/backing with scaling (§D.1)—state it in the module header. Set **`suppressHiDPIScaling`** to **`true`** or **`false`** with a one-line rationale (crisp pixels vs smooth scaling). Gameplay code must not hardcode canvas pixel sizes except via this module (**`.cursor/plans/project-implementation-deep-dive.md`** §D.3).

## Acceptance Criteria

1) `src/engine.ts` (or closure-documented path) has a module header (or block comment) naming the **viewport policy** and **`suppressHiDPIScaling`** choice with rationale. 2) A **`DisplayMode`** enum member appears in source (not only inferred). 3) `rg` finds no `800x600`-style numeric canvas literals in `src/` **outside** this engine module (test files under `src/` may use test doubles—say so in closure notes if needed).
