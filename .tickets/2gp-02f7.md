# Retain Vite content-hashed filenames and avoid service workers (prototype)

**Normative:** **`.cursor/plans/project-implementation-deep-dive.md`** §B.2 (hashed assets), §B.3 (avoid service workers), §B.1 / §B.4 (cache realism).

## Design

Do **not** add `vite-plugin-pwa` or register a service worker unless a **follow-up** ticket explicitly needs offline play.

## Acceptance Criteria

1) Two production builds where **application source** changes produce **different** hashed filenames under `dist/assets/` (or document the equivalent Rollup output behavior). 2) App code does **not** call `navigator.serviceWorker.register`. 3) **`README.md`** or **`vite.config.ts`** comment cites §B.2 and adds **one sentence** on §B.1/§B.4 (non-instant global updates).
