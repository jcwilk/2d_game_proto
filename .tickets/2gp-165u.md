# Character walk: sheet-only artifact — no walk_* PNG tiles

Emit **one** spritesheet image (e.g. `sheet.png`) as the primary generated asset for the character walk preset; do **not** require or emit per-frame `public/art/character/walk_*/character.png` tiles for this preset. Game and manifest/sprite-ref must consume the sheet (frame rects / crops) instead of discrete tile files. See current `public/art/character/manifest.json` and `sprite-ref.json` for today’s walk_* layout.

## Acceptance Criteria

- [ ] Pipeline default for this preset writes **one** sheet PNG under the character art out dir (name per preset, e.g. `sheet.png`) and records it in `manifest.json` (or successor contract); **no** required outputs under `walk_0`–`walk_3` for the preset’s normal path.
- [ ] `sprite-ref.json` (or `src/art` types + loader) updated so runtime resolves animation frames from the **sheet + crop rects** (or documented equivalent), not per-tile URLs only.
- [ ] Automated test or documented manual smoke: game loads walk animation from the sheet asset; stale `walk_*` directories under `public/art/character/` may be deleted or left as legacy only if the pipeline and docs no longer require them.
- [ ] `tools/sprite-generation/README.md` documents the sheet-only contract for character walk.
