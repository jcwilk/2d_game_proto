# HUD: bottom draggable stuck orb (pointer+touch), drop hit-test, frame animation

**Depends on slice spec** `specs/drag-stun-hud.md` **(wor-98q7)** — behavior is single-sourced there.

DOM overlay with a **large** hit target near **bottom center**; **layout** must be explicit (e.g. new flex cell in **`#game-controls`**, absolute layer on **`#game-stage`**, etc.) so it does not fight existing chrome (**`specs/viewport-square-dpad-chrome.md`**).

**Interaction:** **`pointerdown`** starts drag (**`setPointerCapture`** on the control); **`pointermove`** moves ghost/floating icon; **`pointerup`** / **`pointercancel`** / **`lostpointercapture`** end drag. On release, map **client → logical/world** using the same approach as other overlays (**`src/ui/screenOverlay.ts`** `logicalGamePointToClientPoint` inverse / canvas bounds + **`VIEWPORT_SIZE`**) and call **`tryApplyStuckAtWorldCoords`** (wor-39p4). Prefer **world-space** hit-test aligned with **`worldPointInNpcBounds`** rather than duplicating monster AABB in screen space unless spec says otherwise.

**Animation:** Frame **0** while idle and while dragging; play frames **1–3** only on **`'ok'`** from the game API; **miss / cooldown / invalid** → no activation frames. Tie total activation duration to asset frame count + chosen FPS (document constant or sync with cooldown UX so the strip can finish).

**Cooldown:** Visually depleted/disabled while recharging; **functional** gate must match game API (early `pointerup` cannot apply if API rejects).

**Input:** Unified **pointer** events for mouse + touch; **`touch-action: none`** on the control; avoid page scroll while dragging. Mirror d-pad a11y: **`role="button"`** + **`aria-label`**, not **title-only**.

**Stacking:** Document **z-index** vs **`#game-canvas-overlay`**, merchant UI, HP bar, monster “!” (**`src/styles.css`** comments).

## Design

Align with **`specs/viewport-square-dpad-chrome.md`** DOM path; coordinate mapping consistent with **`screenOverlay.ts`** / fixed logical viewport (**avoid vague “camera”** unless **`src/camera/math.ts`** is actually wired for gameplay).

## Acceptance Criteria

- Desktop **mouse** and **mobile touch**: drag-drop works; no stuck gestures / scroll fights (within reason).
- **Successful drop only** plays frames 1–3; miss/cooldown/invalid show frame 0 behavior only.
- Ability respects **cooldown** visually and functionally (API is source of truth).
